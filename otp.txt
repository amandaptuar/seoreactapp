Here are the requirements for the backend developer to implement the OTP verification for the registration page:

1. **New API Endpoint: Send OTP**
   - **URL**: `/api/v1/auth/send-otp`
   - **Method**: `POST`
   - **Payload**: `{ "email": "user@example.com" }`
   - **Action**: 
     - Check if the email already exists in the database. If it does, return an error (e.g., 409 Conflict - Email already registered).
     - Generate a 6-digit OTP (One Time Password).
     - Save the OTP in the database or cache (like Redis) with a short expiration time (e.g., 5-10 minutes) associated with the user's email.
     - Send the OTP to the user's email address using the preferred email service (e.g., EmailJS, SendGrid).
   - **Response**: `{ "success": true, "message": "OTP sent to email" }`

2. **New API Endpoint: Verify OTP**
   - **URL**: `/api/v1/auth/verify-otp`
   - **Method**: `POST`
   - **Payload**: `{ "email": "user@example.com", "otp": "123456" }`
   - **Action**:
     - Retrieve the OTP associated with the email from the database/cache.
     - Compare the provided OTP with the stored OTP.
     - Check if the OTP is expired.
     - If valid and not expired, delete or invalidate the OTP from storage to prevent reuse.
   - **Response (Success)**: `{ "success": true, "message": "OTP verified successfully" }`
   - **Response (Failure)**: `{ "success": false, "message": "OTP invalid or expired" }` (HTTP Status 400 or 401)

3. **Frontend Integration Updates (Once APIs are ready)**
   - The frontend currently uses a mocked "123456" OTP for demo purposes.
   - Update `handleVerifyEmail` in `JoinUsPage.jsx` to make a POST request to `/api/v1/auth/send-otp`.
   - Update `handleVerifyOtp` in `JoinUsPage.jsx` to make a POST request to `/api/v1/auth/verify-otp`.
   - Only set `isEmailVerified = true` if the verify OTP API returns success.

4. **Security Considerations**
   - Implement rate-limiting on the send-otp endpoint to prevent abuse and email spam.
   - Restrict the number of failed OTP verification attempts (e.g., lock out after 3-5 failed attempts).
